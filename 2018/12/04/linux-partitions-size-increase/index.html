记录一次在 Linux 下增加分区大小的过程

2018-12-04

最近因为要开发一个 Android 项目，因此需要在系统上安装 AndroidStudio，安装途中却提示我 java.io.IOException: No space left on device，立刻就意识到分区空间不足了，因此进行了该次扩容。而我本身是双系统（Ubuntu 18.04 作为日常开发使用，而 Windows 10 作为游戏娱乐用），因此我要对 Ubuntu 下的分区进行扩容。

如何扩容

  1. 使用 GUI 工具：GParted （本篇文章不进行阐述）

  2. 利用 fdisk 等指令进行扩容

找出空间不足的分区

首先使用 df -h | grep "dev/sda" 指令查看 /dev/sda 以下各个分区占用比

可以得出挂载于 / 的根分区 /dev/sda10 空间严重不足，剩下 96MB。然后我们使用 fdisk -l 指令查询各分区大小及总和

可以看到这个盘大小总和是 232.9GB，而由于双系统的缘故， /dev/sda1 与 /dev/sda2 这两个分区是 Windows 10 的，因此是 NTFS 文件系统。/dev/sda5 为 grub 预留分区，其余则都是 Linux 的 ext4 文件系统。

查询分区挂载情况

确定了是 /dev/sda10 分区空间不足后，我们利用 mount -l | grep "/dev/sda" 指令查看分区实际挂载情况。

透过上面的输出发现，自重装系统之后，我一直犯下了个低级错误，除了 sda7/8/9/10 以外的 /dev/sda3 扩展分区并没有实际挂载点，实际上这个分区本来是应该挂载到 / 上的，当初重装系统之后在引导处分好 /dev/sda3 后却忘记了引导并不会帮你将分区设成开机自动挂载，导致了这里多出了多余的 50G 扩展分区，因此我们要利用这 50G 的空间并且扩容到 /dev/sda10 里去。
